Key Data Relationships

  • Orders → Inventory — items[].sku references inventory SKUs; warehouse + category align for filtering
  • Backlog → Orders — backlog_items.order_id references orders.order_number
  • Backlog → Purchase Orders — purchase_orders.backlog_item_id; backend enriches backlog with has_purchase_order flag
  • Dashboard — aggregates inventory + orders independently, then merges into 5 KPI values
  • Spending — pre-aggregated; not filterable by warehouse/category
  • Warehouses San FranciscoLondonTokyo
  • Categories Circuit BoardsSensorsActuatorsControllersPower Supplies
  • Order statuses DeliveredShippedProcessingBackordered
  • Date range Jan 2025 – Dec 2025; quarterly shortcuts Q1–Q4
